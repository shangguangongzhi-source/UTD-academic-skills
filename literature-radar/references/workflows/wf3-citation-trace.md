# 工作流 3：引文网络追踪

**目的**：从已知高引论文出发，前向追踪施引文献、后向追踪参考文献，发现更多核心文献。

**使用**：[去重引擎](../dedup-engine.md)、[检索策略](../search-strategy.md)

## 流程

### 1. 确认种子论文

用户提供 1-3 篇种子论文（标题 + DOI），或从工作流 1 的结果中选取高引文献。

### 2. 后向追踪（参考文献）

- WebSearch：`"种子论文标题" cited by` 或 Google Scholar 页面的 "相关文章"
- 提取种子论文的参考文献列表中属于核心池的文献

### 3. 前向追踪（施引文献）

- WebSearch：`cited by "种子论文DOI"` 或 Google Scholar "被引用次数" 链接
- 筛选施引文献中发表在核心池期刊的论文

### 4. 验证与分级

同工作流 1 的验证流程。额外注意：被引论文需验证其引用关系确实存在。

### 5. 去重

使用 [去重引擎](../dedup-engine.md)。

### 6. 输出

使用 `scripts/render_report.py` 生成 HTML 报告。报告需额外标注每篇文献与种子论文的引用关系（引用了 / 被引用 / 同被引）。
