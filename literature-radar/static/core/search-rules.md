# 搜索规则与验证流程

## 强制执行规则（不可放宽）

### 规则 1：真实性红线
- 所有输出的文献**必须真实存在**，可通过 Google Scholar、Web of Science、CNKI 等数据库验证
- **严禁编造**作者姓名、论文标题、期刊名称、发表年份、研究结论
- 如果某篇文献无法通过二次搜索确认存在，**必须删除**
- 对每篇文献提供检索验证信息（DOI 或 URL），方便用户核验

### 规则 2：期刊等级硬门槛
文献**仅在 [journal-tiers.md](journal-tiers.md) 定义的期刊池中**方可标注为核心文献。不在池中的文献降级至补充梯队或剔除。

### 规则 3：时间优先级
- **默认范围**：近 5 年（当前年份往前推 5 年）
- 经典奠基性文献（被引次数极高、开创性理论/方法论文）可放宽年限，但**必须标注说明放宽理由**
- 用户指定时间范围时，以用户要求为准

### 规则 4：输出字段
每篇文献必须包含以下字段：
- 作者 + 年份
- 期刊名称 + 期刊等级标签
- 核心研究结论（1-2 句话，基于原文，不可编造）
- 与本主题关联度（高/中/低 + 一句话说明）

### 规则 5：数量控制
- 核心必读：5-10 篇
- 重要参考：5-10 篇
- 补充参考：3-5 篇
- 质量优先，不要贪多

### 规则 6：语言平衡
- 默认英文 70%，中文 30%
- 中国特色问题（国企改革、精准扶贫、中国制度环境等）应提高中文比例
- 用户指定语言偏好时以用户为准

---

## 文献验证流程

对检索到的每篇候选文献执行以下检查：

```
1. 真实性验证：通过 WebSearch 二次确认论文标题、作者、年份、期刊匹配
2. 期刊分级：对照 journal-tiers.md 确定所属梯队
3. 相关性评估：论文核心研究问题是否与用户主题直接相关
4. 质量信号：被引次数、发表年份、作者知名度
```

**验证失败处理**：如果无法确认文献真实性，**立即剔除**，不进入输出。

## 输出方式

### 首选：渲染脚本生成 HTML

使用 `scripts/render_report.py` 渲染最终 HTML 报告。将结构化文献数据以 JSON 格式传入脚本，脚本自动生成自包含 HTML 文件。

JSON 输入格式见 `scripts/render_report.py` 文档。模板样式固定，确保输出质量一致。

### 回退：直接生成 HTML

当渲染脚本不可用时，参照 `static/templates/html-report.md` 中的模板手动生成 HTML。
