# 段落流检查指南

当工作流执行到步骤 6 时，使用本指南对每个段落进行反向大纲检测。

## 核心原则

段落流不是装饰。一个段落"流得好"意味着外部读者能够识别：

- 该段的**单一信息**
- 首句如何**预告**这条信息
- 每个后续句与前句的**逻辑关系**
- 该段如何**服务于**所在章节的论点

## 读者测试

以一个持怀疑态度但公正的外部读者身份，逐段检查：

1. 该段是否只有一条明确的信息？
2. 首句是否直接陈述该段在做什么？
3. 所有关键术语和缩写是否不需要回看上文就能理解？
4. 每句话是否通过以下方式与前句连接：因果、对比、递进、细化、或举例？
5. 是否有句子承载了属于另一个段落的内容？

## 反向大纲法

对章节级段落流检查：

1. 写下该章节的核心论点或主旨句
2. 写下每个段落的首句（主题句）
3. 写下每个段落下的证据或解释
4. 检查：主题句 → 章节论点（是否匹配）
5. 检查：证据 → 主题句（是否支撑）
6. 修改或删除无法干净映射的段落

如果反向大纲很难做，说明该章节有隐藏的结构问题。

## 综述段落常见问题与修复

### 问题 1：一段承载两个任务

**症状**：一段里同时"阐述共识"又"指出缺口"。

**修复**：在共识段落末尾停止。用新一段专门讨论缺口。

### 问题 2：首句不是主题句

**症状**：首句是背景铺垫或引用，读者不知道这段要说什么。

**修复**：重写首句为 claim-first。背景信息放在第二句。

### 问题 3：句子间缺乏显式连接

**症状**：句子之间是"信息孤岛"，读者需要自己脑补逻辑关系。

**修复**：用显式连接词替换隐含逻辑。

| 隐含逻辑 | 显式连接词 |
|---|---|
| 因果 | 因此、这说明、as a result |
| 对比 | 然而、与此不同、in contrast |
| 递进 | 进一步地、building on this |
| 举例 | 例如、具体而言、for instance |
| 细化 | 更具体地说、从机制上看 |

### 问题 4：文献罗列段

**症状**：`张三（2020）发现A；李四（2021）发现B；王五（2022）发现C。`

**修复**：用一个综述句概括共识/趋势，再用引用支撑。

## 修复操作清单

- [ ] 拆分承载两个任务的段落
- [ ] 确保定义出现在术语首次使用之前
- [ ] 替换模糊过渡为显式逻辑连接词
- [ ] 段落首句改为 claim-first（除非章节需要简短铺垫）
- [ ] 删除无法映射到章节论点的段落或重写其主题句
