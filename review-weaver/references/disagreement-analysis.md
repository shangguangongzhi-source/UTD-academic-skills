# 分歧分析深度指南

当工作流执行到步骤 7 时，使用本指南进行分歧与缺口审计。

## 为什么分歧是综述的灵魂

分歧 = 研究缺口的前置信号。没有分歧讨论的综述缺乏深度，评审会质疑作者的批判性思维。分歧原因分析越具体，越能自然引出研究创新点。

## 分歧的五维分析框架

分析文献分歧时，从以下五个维度中选择适用的（通常 2-3 个就够了）：

### 1. 样本差异

| 检查点 | 示例 |
|---|---|
| 国家/地区差异 | 发达国家 vs 发展中国家 |
| 行业差异 | 制造业 vs 服务业 vs 高科技行业 |
| 企业规模差异 | 大企业 vs 中小企业 |
| 时间窗口差异 | 不同经济周期阶段 |

### 2. 方法差异

| 检查点 | 示例 |
|---|---|
| 识别策略 | OLS（相关）vs DID/IV/RDD（因果） |
| 数据层级 | 截面数据 vs 面板数据 |
| 变量度量 | 不同的核心变量代理指标 |
| 样本选择 | 全样本 vs 特定子样本 |

### 4. 情境差异

### 5. 理论视角差异

## 分歧分析的写作模式

```
[明确矛盾] XX et al. (20XX) 发现 A 对 B 有显著正向影响，
然而 YY et al. (20XX) 在不同情境下得到了相反结论。

[分析原因] 这种分歧可能源于以下两方面：第一，[原因维度 1]，
XX 的样本来自[发达国家]，而 YY 聚焦于[发展中国家]，制度环境的
差异可能导致效应方向不同；第二，[原因维度 2]，XX 使用了
[OLS]方法仅能识别相关关系，而 YY 采用[DID]设计提供了因果推断。

[判断权衡] 总体而言，[哪方证据更可信/在什么条件下哪方成立]。
```

## 分歧→缺口的映射逻辑

```
分歧表现 → 分歧原因 → 分歧尚未解决的问题 → 本研究的切入点
```

## 审计清单

- [ ] 是否有明确的分歧讨论？
- [ ] 分歧是否用对比句式明确呈现？
- [ ] 分歧原因是否具体？
- [ ] 分歧双方是否都有代表性文献支撑？
- [ ] 分歧是否自然过渡到研究缺口？
- [ ] 缺口是否与用户的创新点对接？
