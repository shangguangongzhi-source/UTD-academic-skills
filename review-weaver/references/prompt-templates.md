# 推文内容模板

供推文使用。包含痛点模块、可复制提示词和实操要点。

---

## 痛点模块

> 综述写成流水账，一段接一段地"XX认为……YY发现……"，读完不知道作者想表达什么。
>
> 更深层的问题：写综述前没有想清楚"这篇综述到底要论证什么"，直接上手堆文献，结果是形散神也散。

---

## 可复制提示词

### 子 Prompt A：分假设观点整合

```text
你是一个文献综述写作助手。基于以下精读报告，按我的研究假设归类文献，生成综述主体。

输入：
- 精读报告：[技能2产出的reading-report.html内容或文字版]
- 我的研究主题：[论文题目]
- 我的核心假设：
  1. [假设1]
  2. [假设2]

综述边界：
- 时间范围：[如 2015-2025]
- 学科范围：[如 创新经济学]
- 纳入标准：[如 仅限顶刊+高被引]

要求：
1. 动笔前先写一句话综述论证
2. 以假设为骨架，每条假设下按"共识→分歧→研究缺口"结构组织
3. 有述有评，禁止简单罗列
4. 顶刊观点优先锚定
5. 每个假设末尾用过渡句引出我的研究
6. 保留引用标注，首次出现标注期刊等级
7. 生成完整的 HTML 综述报告
8. 附带综述元数据
9. 总篇幅 3000-5000 字
```

### 子 Prompt C：段落流自检

```text
你是一个学术写作审校助手。对以下综述段落执行段落流检查。

检查项：
1. 该段是否只有一条明确的信息？
2. 首句是否直接预告了该段的内容？
3. 每句话是否与前句有显式逻辑连接？
4. 是否存在文献罗列？
5. 评述是否体现学术判断力？
```

## 实操要点

### 1. 动笔前先想清楚一句话论证

### 2. 分歧点是综述灵魂

### 3. 顶刊观点前置

### 4. 综述边界要显式声明

### 5. 段落流自检是最后的保险

### 6. 与前两个技能的衔接

- 技能 1（文献雷达）→ 筛选出核心文献清单
- 技能 2（精读拆解）→ 产出 8 维度精读报告
- 技能 3（观点织网）→ 基于精读报告生成综述框架
- 三个技能构成完整的文献综述生产链
