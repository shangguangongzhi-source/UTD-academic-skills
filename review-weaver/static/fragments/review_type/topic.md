# 综述类型：主题驱动

以子主题/机制/方法维度为组织线索，适用于没有明确假设、需要对一个领域进行全面梳理的场景（如 standalone 综述文章、研究领域的总体评述）。

## 论证链

`领域范围界定 → 子主题分组 synthesis → 各主题共识与分歧 → 跨主题缺口图谱 → 前沿展望`

## 结构规范

### 横向模块（80%-85%）

按子主题分块，每个子主题对应一个独立区块：

```
二级标题：[子主题名称]

1. 核心概念与理论框架（1-2段）
   - 该子主题的界定、核心概念、主要理论视角

2. 代表性发现与共识（2-3段）
   - 按机制或结论分组（非按年份或作者罗列）
   - 顶刊观点优先锚定
   - 每段一个信息点，有述有评

3. 争议与局限（1-2段）
   - 结论矛盾的维度
   - 方法或样本上的局限
   - 与该子主题相关的研究空白

4. 与其他子主题的衔接（1段）
   - 该子主题与前后子主题的逻辑关系
   - 跨子主题的交叉空白
```

### 纵向模块（15%-20%）

与假设驱动型一致，按时间/方法演进梳理发展轨迹。

## 起草规则

- **按技术主题和机制分组，不按发表年份或作者分组**
- 每个子主题结束时，其局限必须能与用户的贡献建立联系。如果某个子主题的局限与用户研究无关，该子主题可能不属于综述范围
- 使用显式逻辑连接词：`in contrast` / `building on this` / `the remaining disagreement is` / `与之不同` / `在此基础上`
- 避免两种极端：不贬低前人工作，也不一味吹捧。客观陈述前人展示了什么、其范围在哪里
- 子主题数量建议 2-4 个，每个子主题需要足够的文献支撑

## 与假设驱动型的区别

| 维度 | 假设驱动 | 主题驱动 |
|---|---|---|
| 组织线索 | 用户假设 | 子主题/机制/方法 |
| 缺口定位 | 每个假设末尾引出 | 跨子主题空白图谱 |
| 适用场景 | 实证论文的文献综述部分 | standalone 综述文章 |
