# 综述类型：方法学综述

以方法论类别为组织线索，比较不同方法的适用场景、优势和局限，适用于方法论评估、方法选择论证、工具综述。

## 论证链

`方法学问题界定 → 方法分类 → 各方法的优势/局限/适用场景 → 方法演进驱动力 → 本研究的方法选择理由`

## 结构规范

### 横向模块（80%-85%）

按方法论类别分块：

```
二级标题：[方法类别名称]

1. 方法原理与核心假设（1-2段）
   - 方法的基本原理、识别策略、核心假设
   - 适用条件与前提假设

2. 代表性应用与效果（2-3段）
   - 该方法在目标领域的代表性应用
   - 效果评估（识别精度、因果可信度等）
   - 顶刊应用优先展示

3. 局限与争议（1-2段）
   - 方法自身的内在局限（如假设过强、数据要求高等）
   - 应用中的常见问题（如误用、过度解读）
   - 与替代方法的对比

4. 演进趋势（1段）
   - 该方法的改进方向
   - 与新兴方法的融合趋势
```

### 纵向模块（15%-20%）

重点放在方法论演进逻辑：

```
二级标题：[领域]研究方法论演进

1. 早期方法阶段
   - 主流方法、代表性研究、方法论的初始局限

2. 方法革新阶段
   - 新方法的引入动因（旧方法解决不了什么问题）
   - 方法论的标志性进步
   - 代表性方法论论文献

3. 当前方法论前沿
   - 最新方法工具（如机器学习辅助、文本分析、RD 设计等）
   - 多方法 triangulation 的趋势
   - 本研究方法选择在前沿脉络中的位置
```

## 起草规则

- 每种方法的讨论必须包含：原理、应用、局限、与替代方法的对比
- 方法比较时，使用具体的技术维度（如识别策略、假设条件、数据需求、估计精度），不用模糊的"更好/更先进"
- 评价方法时客观中立——不贬低旧方法，也不过度推崇新方法
- 方法学综述的终极目标是为用户的方法选择提供论证基础
- 引用密度可比其他综述类型略低，但要保证每种方法都有足够的代表性引用

## 与其他综述类型的区别

| 维度 | 假设驱动 | 主题驱动 | 方法学 |
|---|---|---|---|
| 组织线索 | 假设 | 子主题 | 方法类别 |
| 核心任务 | 引出研究缺口 | 梳理领域全貌 | 论证方法选择 |
| 分歧讨论 | 结论矛盾 | 结论+理论矛盾 | 方法优劣对比 |
| 纵向重点 | 时间/问题演进 | 时间/问题演进 | 方法论演进 |
